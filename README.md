# Аиша — Telegram-бот для абитуриентов

## Описание

Аиша — MVP Telegram-бота на Python + aiogram 3 для помощи абитуриентам. Бот спокойно собирает регион, баллы ЕГЭ, интересующее направление и тип обучения, отправляет запрос в backend API или временную backend-заглушку и показывает подходящие варианты вузов в виде понятных карточек.

Также в боте есть отдельный раздел психологической поддержки: он помогает уменьшить перегруз от выбора, разложить поступление на маленькие шаги и безопасно реагирует на сообщения о самоповреждении.

Старый C++ Telegram-модуль сохранён как архив в `archive/cpp_telegram_module/`.

## Возможности MVP

- подбор вузов по региону, баллам, направлению и типу обучения;
- пошаговый сценарий через FSM;
- backend-заглушка `backend_stub` с HTTP endpoint `/api/universities`;
- расширенная демонстрационная база по нескольким регионам и направлениям;
- карточки результатов с городом, программой, категорией, предметами, минимальным баллом, формой, сроком, типом и ссылкой;
- категории подбора: безопасные, реалистичные и амбициозные варианты;
- локальное JSON-хранилище пользовательского профиля и последнего подбора;
- профиль пользователя и избранные вузы;
- сохранение вариантов из последнего подбора в избранное;
- сравнение вузов из последнего подбора или избранного;
- главное меню с кнопками;
- психологическая поддержка с готовыми безопасными ответами;
- фиксированный safety-ответ на сообщения про самоповреждение или опасность;
- OpenAI API для короткого объяснения результатов и мягких ответов, если задан `OPENAI_API_KEY`;
- fallback без OpenAI: бот не падает и продолжает работать обычными ответами.

## Архитектура

Основной поток подбора:

```text
Пользователь -> Telegram Bot -> backend_stub/API -> JSON/данные -> Telegram Bot -> Пользователь
```

AI-объяснения и поддержка:

```text
Telegram Bot -> OpenAI API -> мягкое объяснение/поддержка -> Telegram Bot
```

Бот не работает напрямую с базой данных. В MVP вместо настоящего backend и БД используется `backend_stub` с демонстрационным JSON-файлом.

Демонстрационная база содержит регионы: Адыгея, Москва, Краснодарский край, Санкт-Петербург, Татарстан, Крым, Ростовская область, Свердловская область.

Основные направления в MVP: IT, психология, медицина, экономика, юриспруденция, педагогика, дизайн, строительство, туризм.

```bash
python3 -m venv .venv

```

## Настройка `.env`

Создайте локальный `.env`:

```bash
cp .env.example .env
```

Пример:

```env
TELEGRAM_BOT_TOKEN=your_telegram_bot_token_here
OPENAI_API_KEY=your_openai_api_key_here
BACKEND_BASE_URL=http://localhost:8000
```

`TELEGRAM_BOT_TOKEN` обязателен для запуска бота.

`OPENAI_API_KEY` можно оставить пустым. В этом случае бот покажет карточки вузов и fallback-ответы без AI-объяснений.

Настоящий `.env` не должен попадать в GitHub.

## OpenAI-интеграция

OpenAI API подключается через официальный Python SDK `openai`. Ключ берётся только из локального `.env`:

```env
OPENAI_API_KEY=your_openai_api_key_here
```

Если `OPENAI_API_KEY` пустой, бот работает через заранее прописанные fallback-ответы: подбор вузов, меню, поддержка и safety-фильтр продолжают работать без AI.

Если OpenAI API временно недоступен или вернул ошибку, бот не падает, логирует ошибку в консоль и использует обычный fallback.

Safety-сообщения про самоповреждение, суицидальные мысли или опасность для себя не отправляются в OpenAI. В таких случаях бот сразу отвечает фиксированным кризисным текстом.

OpenAI используется только для мягких объяснений результатов подбора, категорий вариантов и аккуратных ответов поддержки. Реальные ключи нельзя хранить в GitHub.

Технически OpenAI вынесен в `telegram_bot/services/ai.py`, используется `AsyncOpenAI`. Если API временно недоступен, исключение логируется, а пользователь получает обычный fallback-ответ.

## Демонстрационная база вузов

Файл `backend_stub/data/universities.json` содержит тестовые программы для проверки MVP. В записях есть название вуза, город, регион, программа, направление, предметы, минимальный балл, тип обучения, стоимость, ссылка, а у новых записей также форма обучения, срок и пометка.

Проходные баллы и ссылки в backend-заглушке условные. Они нужны только для демонстрации логики бота. Для финальной версии нужна настоящая база вузов или production backend API.

## Категории подбора

После поиска Аиша делит варианты на три понятные категории:

- 🟢 Безопасный вариант — минимальный балл ниже твоего с заметным запасом;
- 🟡 Реалистичный вариант — минимальный балл близко к твоему;
- 🔴 Амбициозный вариант — немного не хватает, но можно рассмотреть как цель.

Недоступные варианты, где минимальный балл выше твоего больше чем на 20 баллов, не показываются в основной выдаче.

Категории можно посмотреть через команду `/categories` или кнопку “Как читать категории”.

Важно: это демонстрационная оценка MVP, а не гарантия поступления. Реальные проходные баллы меняются каждый год, поэтому финальное решение нужно сверять с официальными сайтами вузов.

## Профиль и избранное

Бот сохраняет данные последнего подбора локально:

- Telegram ID;
- регион;
- баллы;
- направление;
- тип обучения;
- последние результаты;
- избранные вузы.

Пользователь может открыть “Мой профиль”, увидеть сохранённые данные, количество результатов последнего подбора и количество избранных вузов.

После подбора бот показывает кнопки “Сохранить 1”, “Сохранить 2”, “Сохранить 3”. Сохранённые варианты доступны в разделе “Избранные вузы”.

Избранное хранится локально в `telegram_bot/storage/user_data.json`. Этот файл добавлен в `.gitignore` и не должен попадать в Git. Для production-версии локальный JSON лучше заменить на полноценную базу данных.

## Сравнение вузов

Сравнение доступно через команду `/compare` или кнопку “Сравнить вузы”.

Бот может сравнить:

- последние результаты подбора;
- избранные вузы.

В сравнении показываются карточки выбранных вузов и короткий вывод:

- самый безопасный по баллам вариант;
- самый амбициозный по баллам вариант;
- запас или нехватка баллов, если в профиле сохранены баллы пользователя;
- самый дешёвый платный вариант, если среди выбранных есть стоимость;
- напоминание, что данные демонстрационные и их нужно сверять с официальными сайтами.

Если задан `OPENAI_API_KEY`, OpenAI может добавить короткое пояснение простым языком. Без OpenAI сравнение всё равно работает через обычную локальную аналитику.

Избранное можно редактировать: после открытия “Избранные вузы” бот показывает кнопки `Удалить 1`, `Удалить 2`, `Удалить 3` для первых сохранённых вариантов. Если нужно убрать всё сразу, есть кнопка “Очистить избранное”.

## Тесты

В проекте есть простые pytest-тесты:

- `tests/test_validation.py` — проверка баллов ЕГЭ;
- `tests/test_safety.py` — проверка crisis/safety-фраз;
- `tests/test_backend_filtering.py` — проверка фильтрации backend-заглушки;
- `tests/test_universities_data.py` — проверка структуры и покрытия демонстрационной базы;
- `tests/test_search_cards.py` — проверка форматирования карточек поиска;
- `tests/test_ai_fallback.py` — проверка fallback-режима без OpenAI-ключа;
- `tests/test_user_data.py` — проверка профиля, последнего подбора и избранного;
- `tests/test_recommendation.py` — проверка категорий подбора;
- `tests/test_compare.py` — проверка аналитики и форматирования сравнения.

Запуск:

```bash
pytest
```

Тесты не требуют Telegram-токена, настоящего `.env`, реального OpenAI API или запущенного Telegram-бота.

## Психологическая поддержка и безопасность

Раздел `/support` помогает пользователю выбрать мягкий следующий шаг, если поступление вызывает тревогу, страх ошибки или давление со стороны ожиданий.

Бот не должен:

- ставить диагнозы;
- давать медицинские советы;
- обещать поступление;
- давить на пользователя;
- шутить над тревогой;
- писать агрессивно.

Если пользователь пишет про самоповреждение, суицидальные мысли или опасность для себя, бот не обращается к OpenAI и отвечает фиксированным текстом:

```text
Мне важно, чтобы ты сейчас был в безопасности. Пожалуйста, обратись к взрослому рядом, родственнику, учителю или в экстренную службу. Если есть риск, что ты навредишь себе, позвони 112.
```

## Важно

Данные в `backend_stub/data/universities.json` демонстрационные. Это не официальные проходные баллы и не рекомендация к поступлению. Для финальной версии данные должны приходить из настоящей базы вузов или production backend API.
