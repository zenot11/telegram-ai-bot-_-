# Аиша - Telegram-сервис для абитуриентов

## Описание проекта

“Аиша” — расширенный демонстрационный прототип Telegram-сервиса для абитуриентов на Python + aiogram 3. Бот помогает спокойно подобрать варианты вузов по региону, баллам ЕГЭ, направлению и типу обучения, разделяет варианты на безопасные, реалистичные и амбициозные, сохраняет избранное и позволяет сравнивать несколько программ.

В проекте также есть Telegram Mini App на HTML, CSS и JS. Интерфейс Mini App приведён к стилю общего сайта команды “Аиша”: крупный hero-блок, анимированный бренд Aisha в верхней панели, карточка “Привет, я Аиша!”, тёмно-синие кнопки, вкладки, быстрые демо-сценарии, контекстные подсказки “Aisha советует”, локальные фильтры, визуальное сравнение вузов, персональный план поступления, экспорт отчёта, обратная связь, избранное в браузере, светлая и тёмная тема. Mini App проверяет Telegram WebApp-сессию через backend и синхронизирует избранное с ботом только после подтверждённого `initData`. OpenAI подключён опционально и используется только как дополнительный слой для мягких объяснений и поддержки в Telegram-боте.

По умолчанию используются демонстрационные JSON-данные. На 37 этапе добавлен PostgreSQL-режим для базы вузов: при `USE_POSTGRES=true` backend читает каталог из PostgreSQL по `DATABASE_URL`, но Telegram-бот и Mini App продолжают обращаться к тому же `/api/universities` и получают тот же формат ответа.

Старый C++ Telegram-модуль сохранён как архив в `archive/cpp_telegram_module/`.

## Архитектура

Основной поток подбора:

```text
Пользователь -> Telegram Bot -> backend_stub/API -> JSON/PostgreSQL -> Telegram Bot -> Пользователь
```

Mini App использует тот же endpoint:

```text
Mini App -> backend_stub/API -> JSON/PostgreSQL -> Mini App
```

OpenAI используется отдельно:

```text
Telegram Bot -> OpenAI API -> мягкое объяснение/поддержка -> Telegram Bot
```

Safety-фильтр стоит до OpenAI: сообщения про самоповреждение, суицидальные мысли или опасность для себя обрабатываются локально и не отправляются в OpenAI.

## Временная демонстрационная база вузов

По умолчанию используется демонстрационная база `backend_stub/data/universities.json`. Она нужна для показа логики поиска, категорий, избранного, сравнения и Mini App без настройки внешней БД.

PostgreSQL можно включить отдельно через `USE_POSTGRES=true`. Подробный порядок применения SQL-файлов из `finalproj.zip` описан в [docs/POSTGRES.md](docs/POSTGRES.md).

База содержит несколько регионов и направлений:

- регионы: Адыгея, Москва, Краснодарский край, Санкт-Петербург, Татарстан, Крым, Ростовская область, Свердловская область;
- направления: IT, психология, медицина, экономика, юриспруденция, педагогика, дизайн, строительство, туризм.

Проходные баллы, стоимость и ссылки являются демонстрационными. Это не официальные данные и не гарантия поступления.

Формальный контракт базы описан в [docs/DATA.md](docs/DATA.md). Перед заменой демонстрационного файла на финальную базу нужно выполнить:

```bash
python scripts/check_data.py
```

Ожидаемая структура записи:

```json
{
  "university": "Название вуза",
  "city": "Город",
  "region": "Регион",
  "program": "Программа",
  "direction": "Направление",
  "subjects": ["предмет 1", "предмет 2"],
  "min_score": 200,
  "type": "бюджет",
  "price": null,
  "url": "https://example.ru",
  "study_form": "очная",
  "duration": "4 года",
  "note": "демонстрационные данные"
}
```

Поля `study_form`, `duration`, `note` и `price` могут отсутствовать или быть пустыми. Код должен показывать аккуратные fallback-значения, а не технические `None` или `null`.

## Как заменить данные перед сдачей

Перед сдачей можно оставить JSON fallback и заменить временный файл `backend_stub/data/universities.json`, сохранив структуру полей. Альтернативно можно поднять PostgreSQL по `docs/POSTGRES.md`. В обоих режимах основной endpoint `/api/universities` сохраняет прежний контракт.

После замены файла нужно выполнить:

```bash
python scripts/check_data.py
bash scripts/check_project.sh
```

И вручную проверить несколько сценариев подбора в Telegram и Mini App.

## Настройка `.env`

Создайте локальный `.env`:

```bash
cp .env.example .env
```

Пример:

```env
TELEGRAM_BOT_TOKEN=your_telegram_bot_token_here
OPENAI_API_KEY=your_openai_api_key_here
BACKEND_BASE_URL=http://localhost:8000
WEBAPP_URL=
USE_POSTGRES=false
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/tgbot
USE_POSTGRES_TESTS=false
```

`TELEGRAM_BOT_TOKEN` обязателен для запуска Telegram-бота.

`OPENAI_API_KEY` можно оставить пустым. Тогда бот работает через fallback.

`WEBAPP_URL` можно оставить пустым. Тогда кнопка `Mini App` в меню остаётся обычной текстовой кнопкой и показывает подсказку, а обычный бот работает как раньше.

`USE_POSTGRES=false` оставляет JSON fallback. Для PostgreSQL-режима задайте `USE_POSTGRES=true` и `DATABASE_URL`, как описано в `docs/POSTGRES.md`.

Настоящий `.env` не должен попадать в Git.

## Telegram Mini App

Mini App находится в папке `mini_app/` и остаётся простым: HTML + CSS + JS без React, Vite, Next.js и других frontend-фреймворков.

Визуально Mini App приведён к стилю общего сайта команды “Аиша”: серо-бежевый светлый фон, тёмно-синий основной цвет, анимированный бренд Aisha в верхней панели, крупный заголовок “Построй будущее уже сегодня”, карточка “Привет, я Аиша!”, аккуратная форма подбора и карточки результатов. Также есть тёмная тема для Telegram WebView и обычного браузера.

В Mini App есть вкладки:

- `Главная` — презентационный экран;
- `Подбор` — форма и результаты;
- `Фильтры` — локальная фильтрация последней выдачи по категориям и типу обучения;
- `Сравнение` — таблица сравнения 2–3 выбранных вузов;
- `Избранное` — варианты, сохранённые локально или синхронизированные с Telegram-ботом;
- `План` — персональный план действий по последнему подбору;
- `Экспорт` — отчёт по подбору, избранному, сравнению и плану;
- `Поддержка` — форма обратной связи;
- `О проекте` — объяснение демонстрационных данных и категорий.

На главной и во вкладке `Подбор` есть быстрые сценарии, которые заполняют форму и сразу запускают поиск. Форма содержит короткие подсказки под полями, кнопку очистки и toast-уведомления для действий пользователя: поиск, избранное, сравнение, переключение темы и ошибки backend.

Блок `Aisha советует` строит короткие подсказки локально по текущему состоянию Mini App: есть ли подбор, результаты, избранное, сравнение и какая вкладка открыта. Подсказки можно скрыть; выбор хранится в `localStorage` по ключу `aisha_hints_hidden`.

Вкладка `План` строится локально по `lastResults`, избранному и выбранным вариантам сравнения. Она показывает краткий итог, чеклист готовности, безопасные/реалистичные/амбициозные варианты, следующие действия и кнопку копирования текстового плана.

Вкладка `Экспорт` готовит отчёт по текущему состоянию Mini App: параметры подбора, найденные варианты, избранные вузы, сравнение и план поступления. Отчёт можно скопировать как plain text или отправить на печать, чтобы сохранить PDF через системный диалог браузера. Backend для этого не нужен.

Фильтры и сравнение Mini App работают по уже полученным результатам и не отправляют новый запрос к backend. Mini App отдельно проверяет WebApp-сессию через `/api/webapp/session`: в Telegram-режиме backend валидирует `initData`, а в обычном браузере показывает локальный режим. Избранное синхронизируется с Telegram-ботом только после подтверждённой Telegram-сессии. В обычном браузере без `initData` избранное работает локально через `localStorage`. Выбранные варианты для сравнения и тема тоже сохраняются локально в браузере через `localStorage`.

Локальная проверка:

```bash
source .venv/bin/activate
python -m backend_stub.main
```

Затем открыть в браузере:

```text
http://localhost:8000/miniapp
```

Mini App обращается к:

```text
/api/universities?region=...&score=...&direction=...&type=...&limit=5
```

Для диагностики режима открытия Mini App обращается к:

```text
/api/webapp/session
```

Telegram-бот и Mini App работают с одной backend-заглушкой и одной демонстрационной базой, поэтому сценарии подбора выглядят согласованно в чате и в веб-интерфейсе.

OpenAI и токены в Mini App не используются.

## Проверка через ngrok

Для открытия Mini App внутри Telegram нужен публичный HTTPS URL. Локальный `http://localhost:8000/miniapp` подходит для проверки в браузере, но обычно не откроется на телефоне внутри Telegram.

Пример настройки через публичный HTTPS-туннель:

```env
WEBAPP_URL=https://your-public-url/miniapp
```

После изменения `WEBAPP_URL` нужно перезапустить Telegram-бота. Ngrok/cloudflared не являются зависимостями проекта и не настраиваются автоматически.

## OpenAI

OpenAI подключён через официальный Python SDK `openai`, используется `AsyncOpenAI`.

Ключ берётся только из локального `.env`:

```env
OPENAI_API_KEY=your_openai_api_key_here
```

Если ключ пустой, бот не делает запросы к OpenAI и работает через fallback. Если OpenAI временно недоступен, ошибка логируется в консоль, а пользователь получает обычный безопасный ответ.

Safety/crisis-сообщения не отправляются в OpenAI.

## Категории подбора

После поиска Аиша делит варианты на категории:

- 🟢 безопасный вариант — минимальный балл ниже твоего с заметным запасом;
- 🟡 реалистичный вариант — минимальный балл близко к твоему;
- 🔴 амбициозный вариант — немного не хватает, но вариант можно рассмотреть как цель.

Недоступные варианты, где минимальный балл выше твоего больше чем на 20 баллов, не показываются в основной выдаче.

Категории помогают спокойно разложить варианты, но не гарантируют поступление.

## Нормализация ввода

Бот поддерживает мягкую нормализацию ввода: регистр, лишние пробелы, частые синонимы и некоторые опечатки в регионах, направлениях и типе обучения.

Например, `Адыгеая → 230 → айти → Бюджет` обрабатывается как `Адыгея → 230 → IT → бюджет`, поэтому подбор, `/summary`, избранное и сравнение продолжают работать.

## Профиль, избранное и сравнение

Бот локально сохраняет:

- Telegram ID;
- регион;
- баллы;
- направление;
- тип обучения;
- последние результаты;
- историю последних 5 подборов;
- избранные вузы.

Пользователь может открыть “Мой профиль”, сохранить варианты в “Избранные вузы”, удалить отдельный сохранённый вуз или очистить избранное.

Команда `/history` показывает последние подборы пользователя: регион, баллы, направление, тип обучения, количество найденных вариантов, категории и топ-3 результата. Через кнопку “Повторить последний подбор” можно снова выполнить последний запрос. Команда `/clear_history` очищает только историю подборов, не удаляя профиль, избранное и последний результат.

## Итог последнего подбора

Команда `/summary` показывает компактную сводку по последнему подбору:

- регион, баллы, направление и тип обучения;
- сколько вариантов найдено;
- сколько вариантов попало в безопасную, реалистичную и амбициозную категории;
- топ-3 найденных варианта;
- сколько вузов сохранено в избранном.

После подбора бот также показывает кнопку “Итог подбора”. Из summary удобно перейти к избранному, сравнению вузов или Mini App.

## Советы по последнему подбору

Команда `/advice` анализирует последний подбор локально, без OpenAI. Бот смотрит на количество вариантов, категории, тип обучения и избранное, а затем предлагает спокойные следующие шаги: сохранить 1–2 вуза, сравнить варианты, проверить сайты вузов, попробовать соседний регион, другое направление или платное обучение как запасной маршрут.

Советы не хранятся в storage и строятся каждый раз на основе последнего подбора.

## Фильтры результатов

Команда `/filters` работает с последними результатами подбора и позволяет быстро показать:

- все варианты;
- 🟢 безопасные;
- 🟡 реалистичные;
- 🔴 амбициозные;
- бюджетные;
- платные.

После фильтрации кнопки `Сохранить 1`, `Сохранить 2` и так далее относятся именно к показанной отфильтрованной выдаче. При этом `last_results` не перезаписывается, поэтому `/summary`, `/history` и `/compare` продолжают работать с полным последним подбором.

## Экспорт результата

Команда `/export` формирует отчёт по последнему подбору. В отчёт входят параметры поиска, краткий итог по категориям, найденные варианты, избранные вузы, короткие советы и предупреждение о демонстрационных данных.

Экспорт доступен в двух форматах:

- показать текстом в Telegram, если отчёт достаточно короткий;
- скачать `.txt` файл.

Файл формируется в памяти и отправляется через Telegram как документ. Постоянные export-файлы в проекте не создаются.

Сравнение доступно через `/compare` или кнопку “Сравнить вузы”. Сравнивать можно последние результаты подбора или избранные вузы. Бот показывает более спокойный и более амбициозный вариант, запас или нехватку баллов, стоимость, предметы, форму и срок обучения, если данные указаны.

Локальный файл `telegram_bot/storage/user_data.json` добавлен в `.gitignore` и не должен попадать в Git.

## Команды

Главное меню Telegram-бота оформлено как карточка: локальный баннер, короткий caption и inline-кнопки. Если Telegram не принимает баннер или файл недоступен, бот показывает обычный текстовый fallback с теми же inline-кнопками.

Меню разделено на основные разделы:

- `Подобрать вуз` — новый подбор;
- `Mini App` — расширенный веб-интерфейс;
- `Мои результаты` — итог, избранное, история, сравнение, фильтры и экспорт;
- `Помощник` — советы, категории и психологическая поддержка;
- `Сервис` — профиль, обратная связь, обращения, приватность и сброс данных;
- `О проекте` — описание, демо-сценарий и BotFather.

Основные:

- `/start` — запуск бота;
- `/menu` — главное меню;
- `/search` — подбор вузов;
- `/summary` — итог последнего подбора;
- `/advice` — советы по последнему подбору;
- `/filters` — фильтры последних результатов;
- `/export` — экспорт последнего подбора;
- `/history` — история последних подборов;
- `/compare` — сравнение вузов;
- `/categories` — как читать категории;
- `/support` — психологическая поддержка;
- `/feedback` — обратная связь по работе сервиса;
- `/my_feedback` — мои обращения;
- `/webapp` — открыть Mini App.

Информация:

- `/about` — о проекте;
- `/demo` — сценарий демонстрации;
- `/privacy` — приватность;
- `/next` — следующий шаг после подбора;
- `/botfather` — настройки BotFather;

Сервис:

- `/clear_history` — очистить историю подборов;

## Поддержка и обратная связь

Команда `/feedback` запускает простой сценарий обращения по работе сервиса: пользователь выбирает тип заявки, пишет сообщение и получает номер вида `AISH-0001`. Команда `/my_feedback` показывает последние 5 обращений пользователя.

Mini App также содержит вкладку `Поддержка`. В обычном браузере обращение сохраняется в локальном storage без Telegram ID, а при открытии через Telegram backend проверяет WebApp `initData` и привязывает заявку к пользователю бота. Обращения хранятся локально в `telegram_bot/storage/feedback.json`; файл создаётся автоматически и не должен попадать в Git.
