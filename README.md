# Аиша — Telegram-бот для абитуриентов

## Описание

Аиша — MVP Telegram-бота на Python + aiogram 3 для помощи абитуриентам. Бот спокойно собирает регион, баллы ЕГЭ, интересующее направление и тип обучения, отправляет запрос в backend API или временную backend-заглушку и показывает подходящие варианты вузов в виде понятных карточек.

Также в боте есть отдельный раздел психологической поддержки: он помогает уменьшить перегруз от выбора, разложить поступление на маленькие шаги и безопасно реагирует на сообщения о самоповреждении.

Старый C++ Telegram-модуль сохранён как архив в `archive/cpp_telegram_module/`.

## Возможности MVP

- подбор вузов по региону, баллам, направлению и типу обучения;
- пошаговый сценарий через FSM;
- backend-заглушка `backend_stub` с HTTP endpoint `/api/universities`;
- карточки результатов с городом, программой, предметами, минимальным баллом, типом и ссылкой;
- локальное JSON-хранилище пользовательского профиля и последнего подбора;
- главное меню с кнопками;
- психологическая поддержка с готовыми безопасными ответами;
- фиксированный safety-ответ на сообщения про самоповреждение или опасность;
- OpenAI API для короткого объяснения результатов и мягких ответов, если задан `OPENAI_API_KEY`;
- fallback без OpenAI: бот не падает и продолжает работать обычными ответами.

## Архитектура

Основной поток подбора:

```text
Пользователь -> Telegram Bot -> backend_stub/API -> JSON/данные -> Telegram Bot -> Пользователь
```

AI-объяснения и поддержка:

```text
Telegram Bot -> OpenAI API -> мягкое объяснение/поддержка -> Telegram Bot
```

Бот не работает напрямую с базой данных. В MVP вместо настоящего backend и БД используется `backend_stub` с демонстрационным JSON-файлом.

## Структура проекта

```text
telegram_bot/
  main.py                  # запуск Telegram-бота
  config.py                # настройки из .env
  handlers/                # команды, меню, поиск, поддержка
  keyboards/               # клавиатуры меню, поиска и поддержки
  services/                # backend API, OpenAI, validation, safety
  states/                  # FSM состояния
  storage/                 # локальное JSON-хранилище

backend_stub/
  main.py                  # временный backend API
  data/universities.json   # демонстрационные данные вузов

archive/
  cpp_telegram_module/     # старый C++ Telegram-модуль
```

## Установка

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## Настройка `.env`

Создайте локальный `.env`:

```bash
cp .env.example .env
```

Пример:

```env
TELEGRAM_BOT_TOKEN=your_telegram_bot_token_here
OPENAI_API_KEY=your_openai_api_key_here
BACKEND_BASE_URL=http://localhost:8000
```

`TELEGRAM_BOT_TOKEN` обязателен для запуска бота.

`OPENAI_API_KEY` можно оставить пустым. В этом случае бот покажет карточки вузов и fallback-ответы без AI-объяснений.

Настоящий `.env` не должен попадать в GitHub.

## OpenAI-интеграция

OpenAI API подключается через официальный Python SDK `openai`. Ключ берётся только из локального `.env`:

```env
OPENAI_API_KEY=your_openai_api_key_here
```

Если `OPENAI_API_KEY` пустой, бот работает через заранее прописанные fallback-ответы: подбор вузов, меню, поддержка и safety-фильтр продолжают работать без AI.

Если OpenAI API временно недоступен или вернул ошибку, бот не падает, логирует ошибку в консоль и использует обычный fallback.

Safety-сообщения про самоповреждение, суицидальные мысли или опасность для себя не отправляются в OpenAI. В таких случаях бот сразу отвечает фиксированным кризисным текстом.

OpenAI используется только для мягких объяснений результатов подбора и аккуратных ответов поддержки. Реальные ключи нельзя хранить в GitHub.

Во втором терминале:

## Проверка

Пример ручного сценария в Telegram:

1. `/start`
2. `Подобрать вуз`
3. `Адыгея`
4. `230`
5. `IT`
6. `Бюджет`

Ожидаемый результат: бот покажет несколько тестовых карточек вузов и подпишет, что это демонстрационный подбор MVP.

## Команды бота

- `/start` — запуск бота;
- `/menu` — главное меню;
- `/help` — помощь;
- `/reset` — сброс введённых данных;
- `/support` — психологическая поддержка;
- `/search` — подбор вузов.

## Главное меню

- Подобрать вуз
- Мои баллы
- Направления
- Регионы
- Психологическая поддержка
- Помощь

## Психологическая поддержка и безопасность

Раздел `/support` помогает пользователю выбрать мягкий следующий шаг, если поступление вызывает тревогу, страх ошибки или давление со стороны ожиданий.

Бот не должен:

- ставить диагнозы;
- давать медицинские советы;
- обещать поступление;
- давить на пользователя;
- шутить над тревогой;
- писать агрессивно.

Если пользователь пишет про самоповреждение, суицидальные мысли или опасность для себя, бот не обращается к OpenAI и отвечает фиксированным текстом:

```text
Мне важно, чтобы ты сейчас был в безопасности. Пожалуйста, обратись к взрослому рядом, родственнику, учителю или в экстренную службу. Если есть риск, что ты навредишь себе, позвони 112.
```

## Важно

Данные в `backend_stub/data/universities.json` демонстрационные. Это не официальные проходные баллы и не рекомендация к поступлению. Для финальной версии данные должны приходить из настоящей базы вузов или production backend API.

## Что можно улучшить дальше

- подключить настоящую базу вузов;
- добавить PostgreSQL или Redis для хранения состояния;
- сделать админ-панель для обновления программ;
- добавить личный кабинет абитуриента;
- расширить OpenAI-сценарии для вопросов о направлениях;
- добавить тесты handlers и backend-фильтрации;
- подготовить деплой.
