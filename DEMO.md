# Демонстрация Telegram-бота “Аиша”

## 1. Что показывает демонстрация

Демонстрация показывает полный MVP-сценарий:

- запуск backend-заглушки;
- запуск Telegram-бота;
- главное меню;
- подбор вузов;
- расширенную демонстрационную базу регионов и направлений;
- категории подбора: безопасные, реалистичные и амбициозные варианты;
- обработку некорректных баллов;
- Telegram Mini App;
- психологическую поддержку;
- safety-ответ;
- fallback без OpenAI;
- опциональную OpenAI-интеграцию при наличии `OPENAI_API_KEY`.

## 2. Подготовка

Настоящий `.env` должен быть создан локально из `.env.example` и не должен попадать в GitHub.

Первый терминал:

```bash
source .venv/bin/activate
python -m backend_stub.main
```

Второй терминал:

```bash
source .venv/bin/activate
python -m telegram_bot.main
```

## 3. Сценарий №1: главное меню

Проверить:

```text
/start
/menu
/help
```

Ожидаемый результат: бот приветствует пользователя, показывает главное меню и список команд.

Ввести:

Важно сказать на демонстрации: данные в `backend_stub/data/universities.json` демонстрационные и не являются официальными проходными баллами.

## 6. Сценарий: категории подбора

Ввести:

```text
Подобрать вуз
Адыгея
230
айти
Бюджет
```

Ожидаемый результат: бот показывает карточки с категориями:

- 🟢 безопасный вариант, если есть заметный запас по баллам;
- 🟡 реалистичный вариант, если проходной балл близко к баллам пользователя;
- 🔴 амбициозный вариант, если немного не хватает баллов.

Дополнительно можно нажать `Как читать категории` или ввести `/categories`.

Важно сказать на демонстрации: категории не гарантируют поступление, потому что реальные проходные баллы меняются каждый год.

Проверить фразу:

```text
не хочу жить
```

Ожидаемый результат: бот возвращает фиксированный безопасный текст с рекомендацией обратиться к взрослому рядом или позвонить 112.

Safety-сообщения обрабатываются локально и не отправляются в OpenAI.

## 10. Сценарий: профиль и избранное

Проверить:

1. Пройти подбор:

```text
Адыгея
230
айти
Бюджет
```

2. Нажать `Сохранить 1`.
3. Открыть `Избранные вузы`.
4. Открыть `Мой профиль`.
5. Проверить, что там отображаются сохранённые данные, количество результатов последнего подбора и количество избранных вузов.
6. Нажать `/reset`.
7. Проверить, что профиль и избранное очищены.

Ожидаемый результат: бот сохраняет выбранный вуз из последнего подбора, показывает его в избранном и очищает все локальные данные после сброса.

## 11. Сценарий: кнопки сохранения

1. Пройти подбор:

```text
Адыгея
230
айти
Бюджет
```

2. Убедиться, что количество кнопок `Сохранить N` соответствует количеству показанных вузов.
3. Если отображаются `Сохранить 4` или `Сохранить 5`, нажать одну из этих кнопок.
4. Открыть `Избранные вузы`.
5. Проверить, что выбранный вуз сохранился.

Ожидаемый результат: бот сохраняет любой показанный вариант из текущей выдачи, а не только первые три.

## 12. Сценарий: сравнение вузов

Через последние результаты:

1. Пройти подбор:

```text
Адыгея
230
айти
Бюджет
```

2. Нажать `Сравнить вузы`.
3. Выбрать `Сравнить последние результаты`.
4. Нажать `Сравнить 1 и 2` или `Сравнить первые 3`.

Ожидаемый результат: бот показывает сравнительную карточку и короткий вывод с безопасным и амбициозным вариантом, запасом или нехваткой баллов.

Через избранное:

1. Пройти подбор.
2. Нажать `Сохранить 1`.
3. Нажать `Избранные вузы`.
4. Нажать `Сравнить вузы`.
5. Если избранных меньше 2, бот объясняет, что нужно сохранить минимум два варианта.

## 13. Сценарий: избранное и сравнение вузов

1. Пройти подбор:

```text
Адыгея
230
айти
Бюджет
```

2. Нажать `Сохранить 1`.
3. Нажать `Сохранить 2`.
4. Открыть `Избранные вузы`.
5. Нажать `Сравнить вузы`.
6. Выбрать `Сравнить избранные вузы`.
7. Проверить, что бот показывает сравнение по баллам, типу обучения, стоимости и предметам.
8. Проверить удаление: нажать `Удалить 1`.

Ожидаемый результат: бот сохраняет избранное, не создаёт дубли, сравнивает варианты и умеет удалить конкретный вуз.

## 14. Сценарий №6: OpenAI

Без ключа:

- `OPENAI_API_KEY` пустой;
- бот работает через fallback;
- карточки вузов показываются;
- `/support` работает готовыми безопасными ответами.

С ключом:

- `OPENAI_API_KEY` задан локально в `.env`;
- после `/search` может появляться короткое AI-объяснение;
- в `/support` обычные ситуации могут получать AI-ответ;
- safety-фразы всё равно не отправляются в OpenAI.

## 15. Сценарий: Telegram Mini App

Локальная проверка:

1. Запустить backend:

```bash
source .venv/bin/activate
python -m backend_stub.main
```

2. Открыть в браузере:

```text
http://localhost:8000/miniapp
```

3. Ввести:

```text
Адыгея
230
айти
Бюджет
```

4. Проверить, что появились карточки вузов.

Проверка через Telegram:

1. Указать `WEBAPP_URL` в `.env`.
2. Перезапустить бота.
3. Открыть `/menu`.
4. Нажать `Открыть Mini App`.
5. Проверить, что открылась веб-страница.

Важно: для Telegram Mini App нужен публичный HTTPS URL. Локальный `localhost` подходит только для проверки в браузере.

## 16. Что важно сказать на защите

- Бот не работает только по ключевым словам: есть сценарная логика и FSM.
- Есть backend API-заглушка, бот не работает напрямую с базой данных.
- Демонстрационная база расширена: несколько регионов и направлений.
- Есть демонстрационный Mini App, который локально открывается через backend_stub.
- Есть локальный профиль пользователя и избранные вузы для MVP.
- Кнопки сохранения соответствуют количеству показанных вариантов.
- Есть сравнение вузов из последних результатов и избранного.
- Есть категории подбора: безопасные, реалистичные и амбициозные варианты.
- OpenAI вынесен в отдельный сервис.
- OpenAI опционален: если ключа нет, fallback сохраняет работоспособность.
- Safety-фильтр обрабатывается локально до OpenAI.
- Данные вузов пока демонстрационные.
