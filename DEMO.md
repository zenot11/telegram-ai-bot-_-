# Демонстрация Telegram-бота “Аиша”

## 1. Что показывает демонстрация

Демонстрация показывает расширенный Telegram-сервис для абитуриентов:

- запуск backend-заглушки;
- запуск Telegram-бота;
- главное меню;
- подбор вузов;
- категории подбора: безопасные, реалистичные и амбициозные варианты;
- обработку некорректных баллов;
- профиль пользователя;
- итог последнего подбора;
- советы по последнему подбору;
- фильтры результатов;
- история подборов;
- избранные вузы;
- сравнение вузов;
- Telegram Mini App;
- психологическую поддержку;
- safety-ответ;
- fallback без OpenAI;
- опциональную OpenAI-интеграцию при наличии `OPENAI_API_KEY`;
- PostgreSQL-режим для базы вузов с JSON fallback;
- backend-справочники регионов, городов, направлений, форм обучения и типов конкурса;
- поиск направлений `/api/directions?q=...` по полному PostgreSQL-справочнику кодов, названий и профилей;
- кастомный Mini App picker направления с dropdown-подсказками и кнопкой очистки выбранного направления;
- аудит использования SQL-схемы команды в `docs/DATABASE_USAGE.md`;
- сохранённый контракт `/api/universities` для Telegram-бота и Mini App.

## 2. Подготовка

Настоящий `.env` должен быть создан локально из `.env.example` и не должен попадать в GitHub.

Первый терминал:

```bash
source .venv/bin/activate
python scripts/check_data.py
python -m backend_stub.main
```

Второй терминал:

```bash
source .venv/bin/activate
python -m telegram_bot.main
```

Проверить:


Ожидаемый результат: бот приветствует пользователя, показывает карточное главное меню с баннером, caption и inline-кнопками. Если баннер не отправился, должен появиться текстовый fallback с теми же разделами.

Проверить навигацию главного меню:

1. На первом экране есть только основные разделы:
   `🎓 Подобрать вуз`, `📱 Mini App`, `📌 Мои результаты`, `🤝 Помощник`, `⚙️ Сервис`, `ℹ️ О проекте`.

8. Открыть `ℹ️ О проекте`: там должны быть описание проекта, демо-сценарий и BotFather.

## 5. Основной сценарий подбора

Ввести:

```text
🎓 Подобрать вуз
Адыгея
230
айти
Бюджет
```

Ожидаемый результат: бот нормализует “айти” в IT-направления, обращается к `backend_stub` и показывает карточки вузов с категориями.

Для проверки устойчивости можно ввести регион с опечаткой:

```text
Адыгеая
230
айти
Бюджет
```

Ожидаемый результат: в JSON fallback бот приведёт регион к `Адыгея`, а в PostgreSQL mode backend также найдёт записи региона `Республика Адыгея`; направление `IT` будет раскрыто в реальные SQL-направления.

Важно сказать на демонстрации:

> Основной пользовательский сценарий рассчитан на PostgreSQL через `USE_POSTGRES=true`; JSON остаётся fallback, чтобы проект запускался без базы и проходил тесты.

Можно также показать, что справочники строятся через backend:

```bash
curl "http://localhost:8000/api/regions"
curl "http://localhost:8000/api/directions?limit=5"
curl "http://localhost:8000/api/directions?q=09.03.04&limit=20"
curl "http://localhost:8000/api/directions?q=архитектура&limit=20"
curl "http://localhost:8000/api/achievements"
```

## 7. Категории подбора

После подбора проверить, что у вариантов есть категории:

- 🟢 безопасный вариант — есть заметный запас по баллам;
- 🟡 реалистичный вариант — баллы близки к проходному уровню;
- 🔴 амбициозный вариант — немного не хватает, но можно рассматривать как цель.

Дополнительно проверить:

```text
/categories
```

Ожидаемый результат: бот объясняет категории и не обещает поступление.

## 11. История подборов

Проверить:

1. Сделать подбор через `/search`.
2. Открыть `/history` или кнопку `🕘 История подборов`.
3. Проверить, что в истории есть регион, баллы, направление, финансирование, количество найденных вариантов и топ-3 результата.
4. Нажать `Повторить последний подбор`.
5. Проверить, что бот снова показывает карточки вузов.
6. Нажать `Очистить историю` или выполнить `/clear_history`.

Ожидаемый результат: история очищается отдельно, без удаления профиля, избранного и последнего подбора.

## 12. Советы по последнему подбору

Проверить:

1. Сделать подбор через `/search`.
2. Открыть `/summary`.
3. Открыть `/advice`.
4. Проверить, что бот предлагает сохранить 1–2 варианта, сравнить вузы через `/compare` и проверить сайты вузов.

Ожидаемый результат: советы строятся локально по последнему подбору, без OpenAI, и помогают выбрать следующий спокойный шаг.

## 13. Кнопки сохранения

Проверить:

1. Пройти подбор.
2. Убедиться, что количество кнопок `⭐ Сохранить N` соответствует количеству показанных вузов.

4. Открыть `⭐ Избранные вузы`.
5. Проверить, что выбранный вуз сохранился.

Ожидаемый результат: бот сохраняет любой показанный вариант из текущей выдачи, максимум до 5.

5. Получить номер заявки вида `AISH-0001`.

7. Проверить, что заявка видна в списке.
8. Убедиться, что `/support` остаётся отдельной психологической поддержкой.

4. Открыть Mini App через Telegram `/webapp`.
5. Отправить обращение в Telegram mode.
6. Вернуться в бот и проверить `/my_feedback`.

Ожидаемый результат: обращения сохраняются локально в `telegram_bot/storage/feedback.json`, пользователь получает номер заявки, а user_id для Telegram mode берётся только из проверенного WebApp `initData`.

## 17. Проверка базы вузов

Перед демонстрацией можно показать, что структура базы проверяется отдельным скриптом:

```bash
python scripts/check_data.py
```

Ожидаемый результат: скрипт показывает количество записей, регионов, направлений и типов обучения для JSON fallback. Для PostgreSQL есть отдельная ручная проверка `python scripts/check_postgres.py`, которая также выводит диагностику по московским программам `09.03.04`, quality audit баллов, форм, конкурсов, регионов, q-search направлений и coverage audit по категориям, регионам, финансированию, формам и квотам. Документ `docs/DATABASE_USAGE.md` можно открыть как доказательство, что SQL-схема команды проверена по таблицам и полям.

## 18. Сравнение вузов

Через последние результаты:

1. Пройти подбор.
2. Нажать `⚖️ Сравнить вузы`.
3. Выбрать `Сравнить последние результаты`.
4. Нажать `Сравнить 1 и 2` или `Сравнить первые 3`.

Через избранное:

1. Пройти подбор.
2. Сохранить минимум два варианта.
3. Открыть `⚖️ Сравнить вузы`.
4. Выбрать `Сравнить избранные вузы`.

Ожидаемый результат: бот показывает сравнение по баллам, запасу или нехватке, типу обучения, стоимости и предметам.

Если у выбранного варианта проходной балл требует уточнения, сравнение пишет `не указан` или `требует уточнения` и не выбирает его как лучший только из-за низкого `min_score`.

## 19. Удаление из избранного

Ожидаемый результат: удаление не ломает профиль и не создаёт дубли.

Safety-сообщения обрабатываются локально и не отправляются в OpenAI.

## 22. OpenAI

Без ключа:

- `OPENAI_API_KEY` пустой;
- бот работает через fallback;
- карточки вузов показываются;
- `/support` работает готовыми безопасными ответами.

С ключом:

- `OPENAI_API_KEY` задан локально в `.env`;
- после `/search` может появляться короткое AI-объяснение;
- в `/support` обычные ситуации могут получать AI-ответ;
- safety-фразы всё равно не отправляются в OpenAI.

```bash
source .venv/bin/activate

```

2. Открыть в браузере:

```text

```

3. Ввести:

```text
Адыгея
230
айти
Бюджет
```

4. Проверить, что появились карточки вузов.

Проверить визуальный стиль Mini App:

- светлый серо-бежевый фон;
- верхняя навигация “Аиша / Главная / Подбор / Фильтры / Сравнение / Избранное / О проекте”;
- hero-блок “Построй будущее уже сегодня”;
- карточка “Привет, я Аиша!”;
- тёмно-синие кнопки;
- светлые карточки результатов.

Ожидаемый результат: Mini App визуально соответствует общему стилю сайта команды “Аиша”, но остаётся простым HTML + CSS + JS без новых зависимостей.

Проверить брендовый блок и подсказки Mini App:

1. Открыть главную вкладку Mini App.
2. Показать анимированный бренд `Aisha` в верхней панели: буквы появляются плавно и не дублируют hero-заголовок.
3. Показать блок `Aisha советует`.
4. Нажать быстрый сценарий `Адыгея · 230 · IT · Бюджет`.
5. Проверить, что подсказка изменилась после подбора.
6. Добавить вуз в избранное и затем добавить 2 вуза к сравнению.
7. Открыть вкладки `План`, `Экспорт` и `Поддержка` и показать, что подсказка меняется по контексту.
8. Нажать `Скрыть`, затем `Показать подсказки`.
9. Обновить страницу и проверить, что скрытие подсказок сохраняется.

Ожидаемый результат: подсказки строятся локально по состоянию Mini App, не используют OpenAI, не мешают работе и поддерживают светлую/тёмную тему.

Проверить новые вкладки Mini App:

1. На главной нажать быстрый сценарий `Адыгея · 230 · IT · Бюджет`.
2. Проверить, что Mini App перешёл во вкладку `Подбор`, заполнил форму, запустил поиск и показал toast.
3. Нажать `Очистить форму` и проверить, что результаты, избранное, сравнение и тема не сбросились.
4. Выполнить подбор вручную:

```text
Адыгея
230
IT
Бюджет
```

5. Проверить, что появился блок `Итог подбора`.
6. Проверить строку `Найдено: N вариантов. Можно сохранить, отфильтровать или сравнить.`
7. Открыть вкладку `Фильтры`.
8. Нажать `Безопасные`, затем `Реалистичные`, затем `Амбициозные`.
9. Нажать `Платное` и убедиться, что пустой фильтр показывает понятное сообщение, если платных вариантов нет.
10. В результатах нажать `В избранное` и увидеть toast.
11. Открыть вкладку `Избранное`.
12. Проверить, что выбранный вуз появился в локальном избранном.
13. Удалить вуз из избранного или нажать `Очистить избранное` и проверить toast.

Проверить picker направления:

1. Открыть вкладку `Подбор`.
2. Нажать поле `Направление`: должен открыться dropdown с подсказкой `Начни вводить код или название`.

4. Нажать `×` внутри поля: очищается только направление, остальные поля остаются.
5. Ввести `архитектура` и выбрать другой вариант.
6. Проверить `ArrowDown`, `ArrowUp`, `Enter` и `Escape`.

Ожидаемый результат: смена направления не требует вручную стирать длинное старое значение, а поиск подсказок продолжает идти через полный PostgreSQL-справочник.

Проверить статус Mini App-сессии:

1. Открыть `http://localhost:8000/miniapp` в обычном браузере.

## 26. Что важно сказать на защите

- Бот не работает только по ключевым словам: есть сценарная логика и FSM.
- Есть backend API-заглушка, бот не работает напрямую с базой данных.
- Основной сценарий PostgreSQL-first: backend может читать каталог из PostgreSQL, а JSON остаётся fallback для локального запуска.
- Mini App и бот используют один API-контракт, поэтому смена источника данных не требует переписывать поиск, сравнение и карточки.
- Mini App локально открывается через `backend_stub`, а в Telegram использует тот же frontend и API.
- Для Mini App внутри Telegram нужен публичный HTTPS URL.
- Есть локальный профиль пользователя и избранные вузы.
- Есть итог последнего подбора через `/summary`.
- Есть советы по последнему подбору через `/advice`.
- Есть фильтры результатов через `/filters`.
- Есть история подборов через `/history`, повтор последнего запроса и отдельная очистка истории.
- Есть сравнение вузов из последних результатов и избранного.
- Есть категории подбора: безопасные, реалистичные и амбициозные варианты.
- OpenAI вынесен в отдельный сервис и остаётся опциональным.
- Safety-фильтр обрабатывается локально до OpenAI.
