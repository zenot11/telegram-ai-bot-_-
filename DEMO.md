# Демонстрация Telegram-бота “Аиша”

## 1. Что показывает демонстрация

Демонстрация показывает расширенный Telegram-сервис для абитуриентов:

- запуск backend-заглушки;
- запуск Telegram-бота;
- главное меню;
- подбор вузов;
- категории подбора: безопасные, реалистичные и амбициозные варианты;
- обработку некорректных баллов;
- профиль пользователя;
- итог последнего подбора;
- избранные вузы;
- сравнение вузов;
- Telegram Mini App;
- психологическую поддержку;
- safety-ответ;
- fallback без OpenAI;
- опциональную OpenAI-интеграцию при наличии `OPENAI_API_KEY`;
- подготовленность проекта к замене демонстрационного JSON на финальные данные перед сдачей.

## 2. Подготовка

Настоящий `.env` должен быть создан локально из `.env.example` и не должен попадать в GitHub.

Первый терминал:

```bash
source .venv/bin/activate
python -m backend_stub.main
```

Второй терминал:

```bash
source .venv/bin/activate
python -m telegram_bot.main
```

## 3. Главное меню и справка

Проверить:

```text
/start
/menu
/help
```

Ожидаемый результат: бот приветствует пользователя, показывает главное меню и список актуальных команд.

## 4. Информационные команды

Проверить:

```text
/about
/demo
/privacy
/next
/botfather
```

Ожидаемый результат:

- `/about` кратко объясняет проект;
- `/demo` показывает сценарий демонстрации;
- `/privacy` объясняет хранение данных и OpenAI fallback;
- `/next` показывает, что подготовить перед финальной сдачей;
- `/botfather` показывает команды и настройки Menu Button/Main App.

## 5. Основной сценарий подбора

Ввести:

```text
Подобрать вуз
Адыгея
230
айти
Бюджет
```

Ожидаемый результат: бот нормализует “айти” в IT, обращается к `backend_stub` и показывает карточки вузов с категориями.

Важно сказать на демонстрации:

> Сейчас в проекте временная демонстрационная база. Перед сдачей её можно заменить на финальный файл с вузами, не переписывая основную логику поиска, сравнения и Mini App.

## 7. Категории подбора

После подбора проверить, что у вариантов есть категории:

- 🟢 безопасный вариант — есть заметный запас по баллам;
- 🟡 реалистичный вариант — баллы близки к проходному уровню;
- 🔴 амбициозный вариант — немного не хватает, но можно рассматривать как цель.

Дополнительно проверить:

```text
/categories
```

Ожидаемый результат: бот объясняет категории и не обещает поступление.

## 8. Ошибки баллов

В сценарии подбора на шаге ввода баллов проверить:

```text
abc
-1
401
```

Ожидаемый результат: бот не падает и просит ввести корректные баллы.

## 9. Профиль и избранное

Проверить:

1. Пройти подбор:

```text
Адыгея
230
айти
Бюджет
```

2. Нажать `Сохранить 1`.
3. Нажать `Сохранить 2`.
4. Открыть `Избранные вузы`.
5. Открыть `Мой профиль`.
6. Проверить, что отображаются регион, баллы, направление, тип обучения, количество результатов и количество избранных вузов.

Ожидаемый результат: бот сохраняет выбранные варианты локально и показывает их без технических значений `None` или `null`.

## 10. Итог последнего подбора

Проверить:

```text
/summary
```

Ожидаемый результат: бот показывает регион, баллы, направление, тип обучения, количество найденных вариантов, категории, топ-3 вариантов и количество избранных вузов.

## 11. Кнопки сохранения

Проверить:

1. Пройти подбор.
2. Убедиться, что количество кнопок `Сохранить N` соответствует количеству показанных вузов.
3. Если отображаются `Сохранить 4` или `Сохранить 5`, нажать одну из этих кнопок.
4. Открыть `Избранные вузы`.
5. Проверить, что выбранный вуз сохранился.

Ожидаемый результат: бот сохраняет любой показанный вариант из текущей выдачи, максимум до 5.

## 12. Сравнение вузов

Через последние результаты:

1. Пройти подбор.
2. Нажать `Сравнить вузы`.
3. Выбрать `Сравнить последние результаты`.
4. Нажать `Сравнить 1 и 2` или `Сравнить первые 3`.

Через избранное:

1. Пройти подбор.
2. Сохранить минимум два варианта.
3. Открыть `Сравнить вузы`.
4. Выбрать `Сравнить избранные вузы`.

Ожидаемый результат: бот показывает сравнение по баллам, запасу или нехватке, типу обучения, стоимости и предметам.

## 13. Удаление из избранного

Проверить:

1. Открыть `Избранные вузы`.
2. Нажать `Удалить 1`.
3. Проверить, что выбранный вуз удалён.
4. Если избранное стало пустым, бот показывает понятный пустой текст.

Ожидаемый результат: удаление не ломает профиль и не создаёт дубли.

## 14. Психологическая поддержка

Проверить:

```text
/support
Мне тревожно
Я не знаю, куда поступать
Я боюсь не поступить
На меня давят родители
Составить короткий план
```

Ожидаемый результат: бот отвечает мягко, без диагнозов, давления и медицинских советов.

## 15. Safety-фильтр

Проверить фразу:

```text
не хочу жить
```

Ожидаемый результат: бот возвращает фиксированный безопасный текст с рекомендацией обратиться к взрослому рядом или позвонить 112.

Safety-сообщения обрабатываются локально и не отправляются в OpenAI.

## 16. OpenAI

Без ключа:

- `OPENAI_API_KEY` пустой;
- бот работает через fallback;
- карточки вузов показываются;
- `/support` работает готовыми безопасными ответами.

С ключом:

- `OPENAI_API_KEY` задан локально в `.env`;
- после `/search` может появляться короткое AI-объяснение;
- в `/support` обычные ситуации могут получать AI-ответ;
- safety-фразы всё равно не отправляются в OpenAI.

## 17. Telegram Mini App

Локальная проверка:

1. Запустить backend:

```bash
source .venv/bin/activate
python -m backend_stub.main
```

2. Открыть в браузере:

```text
http://localhost:8000/miniapp
```

3. Ввести:

```text
Адыгея
230
айти
Бюджет
```

4. Проверить, что появились карточки вузов.

Проверить визуальный стиль Mini App:

- светлый серо-бежевый фон;
- верхняя навигация “Аиша / Главная / Подбор / Категории / О проекте”;
- hero-блок “Построй будущее уже сегодня”;
- карточка “Привет, я Аиша!”;
- тёмно-синие кнопки;
- светлые карточки результатов.

Ожидаемый результат: Mini App визуально соответствует общему стилю сайта команды “Аиша”, но остаётся простым HTML + CSS + JS без новых зависимостей.

Проверка через ngrok или другой HTTPS-туннель:

1. Получить публичный HTTPS URL.
2. Указать в `.env`:

```env
WEBAPP_URL=https://your-public-url/miniapp
```

3. Перезапустить бота.
4. Открыть `/webapp` или `/menu`.
5. Нажать `Открыть Mini App`.

Важно: для Telegram Mini App нужен публичный HTTPS URL. Локальный `localhost` подходит только для проверки в браузере.

## 18. Menu Button

В BotFather можно настроить Menu Button:

```text
URL = WEBAPP_URL из .env
Название = Открыть Аишу
```

Ожидаемый результат: после настройки у пользователя появляется кнопка меню Telegram для открытия Mini App.

## 19. Сброс данных

Проверить:

```text
/reset
```

Ожидаемый результат: профиль, последний подбор и избранное очищены.

## 20. Что важно сказать на защите

- Бот не работает только по ключевым словам: есть сценарная логика и FSM.
- Есть backend API-заглушка, бот не работает напрямую с базой данных.
- Сейчас используется временная демонстрационная база.
- Перед сдачей её можно заменить на финальный файл с вузами, не переписывая основную логику поиска, сравнения и Mini App.
- Есть демонстрационный Mini App, который локально открывается через `backend_stub`.
- Для Mini App внутри Telegram нужен публичный HTTPS URL.
- Есть локальный профиль пользователя и избранные вузы.
- Есть итог последнего подбора через `/summary`.
- Есть сравнение вузов из последних результатов и избранного.
- Есть категории подбора: безопасные, реалистичные и амбициозные варианты.
- OpenAI вынесен в отдельный сервис и остаётся опциональным.
- Safety-фильтр обрабатывается локально до OpenAI.
