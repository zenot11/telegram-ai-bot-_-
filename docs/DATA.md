# Данные вузов

Этот документ описывает формат JSON-базы вузов для проекта “Аиша”.

Сейчас используется демонстрационный файл:

```text
backend_stub/data/universities.json
```

Перед сдачей этот файл можно заменить на финальную базу вузов, если сохранить структуру полей. Telegram-бот, Mini App и endpoint `/api/universities` при этом переписывать не нужно.

## Формат файла

JSON должен быть массивом объектов:

```json
[
  {
    "university": "Название университета",
    "city": "Город",
    "region": "Регион",
    "program": "Название программы",
    "direction": "IT",
    "subjects": ["русский язык", "математика", "информатика"],
    "min_score": 230,
    "type": "бюджет",
    "price": "не указана",
    "url": "https://example.ru",
    "study_form": "очная",
    "duration": "4 года",
    "note": "официальные данные"
  }
]
```

## Обязательные поля

- `university` - строка, название вуза.
- `city` - строка, город.
- `region` - строка, регион.
- `program` - строка, название программы.
- `direction` - строка, направление.
- `subjects` - массив строк.
- `min_score` - целое число, не строка.
- `type` - строка, финансирование.
- `url` - строка, желательно официальный сайт программы или вуза.

## Опциональные поля

- `price` - строка, число или `null`.
- `study_form` - строка, например `очная`.
- `duration` - строка, например `4 года`.
- `note` - строка, например `официальные данные` или `демонстрационные данные`.

## Финансирование

В базе лучше хранить канонические значения:

- `бюджет`
- `платное`

Загрузчик также умеет нормализовать варианты:

- `budget`, `бесплатно` -> `бюджет`
- `paid`, `контракт`, `договор` -> `платное`

## Проверка базы

Перед запуском проекта или перед сдачей нужно выполнить:

```bash
python scripts/check_data.py
```

При успешной проверке:

```text
Data check passed.
Records: 120
Regions: 15
Directions: 8
Types: budget=70, paid=50
```

При ошибках:

```text
Data check failed.
Record #12: missing required field "university"
Record #18: "min_score" must be integer
Record #22: "subjects" must be list of strings
```

## Как подключить финальную базу

1. Подготовить финальный JSON в описанном формате.
2. Заменить файл:

```text
backend_stub/data/universities.json
```

3. Запустить проверку:

```bash
python scripts/check_data.py
```

4. Если есть ошибки, исправить указанные записи.
5. Запустить общую проверку проекта:

```bash
bash scripts/check_project.sh
```

Если структура полей сохранена, основная логика поиска, категорий, сравнения, экспорта и Mini App продолжит работать через тот же backend API.
