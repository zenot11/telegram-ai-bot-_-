# Data compatibility: JSON and PostgreSQL

Этот документ фиксирует совместимость данных после подключения PostgreSQL-источника. Цель - оставить старый контракт `/api/universities` рабочим для Telegram-бота и Mini App, но сделать PostgreSQL основным пользовательским источником при `USE_POSTGRES=true`, а JSON оставить fallback без live-базы.

## Итог аудита

| Поле API | JSON fallback | PostgreSQL mode | Поведение UI |
| --- | --- | --- | --- |
| `university` | есть | берётся из полного `universities.name` | показывается как название вуза |
| `city` | есть | `universities.city` | показывается в карточках |
| `region` | есть | `universities.region` | используется в фильтрах и сравнении |
| `program` | есть | `directions.name` | показывается в заголовке |
| `direction` | есть | `directions.name` | используется в поиске и сравнении |
| `subjects` | список предметов | часто отсутствует, отдаётся как `[]` | строка скрывается, если список пустой |
| `min_score` | число | `passing_scores.min_score` | хранится как исходное число, но `0/1` не показываются как обычный проходной балл |
| `score_is_valid` | может отсутствовать | `true`, если `min_score > 1` | управляет категорией и расчётом запаса |
| `score_display` | может отсутствовать | текст для UI | `185` или `не указан` |
| `score_note` | может отсутствовать | пометка качества балла | `балл требует уточнения` для `0/1` |
| `type` | `бюджет` или `платное` | нормализация `admission_type` | показывается в карточках |
| `price` | число, строка или `null` | сейчас обычно отсутствует | скрывается, если значения нет |
| `url` | ссылка | `universities.website` | показывается только если есть |
| `study_form` | опционально | нормализация enum `full_time/part_time/evening` | показывается только если есть |
| `duration` | опционально | сейчас отсутствует | скрывается, если значения нет |
| `note` | опционально | `passing_scores.note` | показывается только если есть |
| `year` | может отсутствовать | `passing_scores.year` | показывается как `Год данных` |
| `faculty` | может отсутствовать | `faculties.name` | показывается как `Факультет` |
| `admission_type` | может отсутствовать | исходный тип конкурса | показывается как `Конкурс`, если это не простой `budget/paid` |
| `admission_type_label` | может отсутствовать | человекочитаемый тип конкурса | показывается как `Конкурс` |
| `profile` | может отсутствовать | `directions.profile` | используется в поиске и может показываться как деталь программы |
| `direction_code` | может отсутствовать | `directions.code` | служебная деталь направления, старые клиенты игнорируют |
| `university_full_name` | может отсутствовать | `universities.name` | помогает UI не выводить технический `short_name` главным названием |
| `university_short_name` | может отсутствовать | `universities.short_name` | показывается отдельно только если это полезное короткое название |
| `source` | отсутствует | `postgresql` | внутреннее поле, в обычном UI не показывается |

## Правила совместимости

- PostgreSQL является основным пользовательским источником при `USE_POSTGRES=true`.
- JSON fallback не удаляется и используется для тестов, локального запуска без базы и резервного режима.
- PostgreSQL mode возвращает все старые поля, которые нужны существующим клиентам.
- Отсутствующие PostgreSQL-данные нормализуются в безопасные значения: пустой список для `subjects`, пустая строка для текстовых необязательных полей, `null` для `price`.
- Пользовательские поверхности не показывают технические значения `None`, `null`, `undefined`, `[]` и не выводят пустые строки вида `Стоимость:`, `Предметы:` или `Срок:`.
- Поле `source` разрешено в API для диагностики, но скрыто в карточках, сравнении, экспорте, summary и Mini App.
- `min_score=0` и `min_score=1` не удаляются из выдачи, но считаются невалидными для пользовательского расчёта проходного балла.
- Запас и категории safe/realistic/ambitious считаются только по валидному `min_score > 1`.
- Регионы `Крым` и `Адыгея` раскрываются до PostgreSQL-названий `Республика Крым` и `Республика Адыгея`.
- Старое направление `IT` раскрывается в реальные SQL-направления и профили, а не ищется как буквальное `direction = IT`.
- Технические короткие названия вроде `РЦТИ-26` не используются как главное название карточки.

## Индивидуальные достижения

Таблица PostgreSQL `achievements` является общим справочником. В ней нет связи с конкретными университетами, поэтому `/api/achievements` и UI показывают такие записи как общие индивидуальные достижения. Они не считаются университетскими правилами приёмной комиссии и не подмешиваются в карточки вузов как вуз-специфичные баллы.

## Баллы и амбициозный зазор

Поиск сохраняет поведение JSON-прототипа: вариант попадает в выдачу, если `min_score <= score + 20`. Эти 20 баллов - амбициозный зазор, чтобы пользователь видел не только полностью безопасные варианты, но и программы немного выше текущего результата.

Это правило одинаково для JSON и PostgreSQL mode и покрыто тестами.

Малые значения `min_score` из SQL-базы не исправляются кодом. Если в PostgreSQL лежит `min_score=25`, интерфейс показывает его вместе с контекстом `admission_type`, `note` и `year`, чтобы пользователь видел источник смысла, а проект не фальсифицировал данные.

Для `min_score=0` и `min_score=1` интерфейс показывает `Проходной балл: не указан` или пометку `балл требует уточнения`. Такие строки могут относиться к квотам, неполным данным или служебным значениям источника, поэтому они не превращаются в `Запас: +279` и не делают вариант безопасным.
