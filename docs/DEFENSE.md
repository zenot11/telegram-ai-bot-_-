# Сценарий защиты проекта “Аиша”

## Что сказать в начале

Я разработал Telegram-бота “Аиша” для помощи абитуриентам. Бот помогает подобрать вуз по региону, баллам ЕГЭ, направлению и типу обучения, показывает категории поступления, позволяет сохранить интересные варианты, сравнить их и открыть Mini App.

Проект демонстрационный: сейчас используется временная база вузов, а финальные данные можно подставить перед сдачей без переписывания основной логики.

## Что показать

1. GitHub repository.
2. Структуру проекта.
3. README и папку `docs/`.
4. Запуск backend:

```bash
bash scripts/run_backend.sh
```

5. Запуск Telegram-бота:

```bash
bash scripts/run_bot.sh
```

6. `/start`.
7. `/search`.
8. Сценарий:

```text

230
айти
Бюджет
```

9. Показать категории:

- безопасный вариант;
- реалистичный вариант;
- амбициозный вариант.

10. Открыть `/summary`.
11. Открыть `/advice` и показать, что бот объясняет следующие шаги после подбора.
12. Открыть `/filters` и показать безопасные, амбициозные или бюджетные варианты.
13. Открыть `/export` и показать текстовый отчёт или скачать `.txt`.
14. Открыть `/history` и показать повтор последнего подбора.
15. Сохранить 1-2 вуза.
16. Открыть “Избранные вузы”.
17. Открыть `/compare`.
18. Открыть `/webapp`.
19. Показать Mini App.
20. В Mini App открыть вкладки `Подбор`, `Фильтры`, `Избранное` и показать локальный итог подбора.
21. В Mini App открыть вкладку `Экспорт` и показать отчёт по подбору.
22. В Mini App открыть вкладку `Поддержка` и показать форму обращения.
23. Открыть `/feedback`, создать заявку и проверить `/my_feedback`.
24. Открыть `/support`.
25. Аккуратно сказать, что бот умеет мягко реагировать на тревожные фразы и имеет локальный safety-фильтр.
26. Показать тесты:

```bash
bash scripts/check_project.sh
```

## Что сказать про данные

Текущая база вузов демонстрационная и лежит в:

```text
backend_stub/data/universities.json
```

Финальную базу можно подставить в этот файл при сохранении структуры полей. Тогда основной код поиска, сравнения, summary и Mini App менять не нужно.

## Что сказать про OpenAI

OpenAI опционален.

Если `OPENAI_API_KEY` пустой, бот работает через локальный fallback:

- подбор вузов работает;
- сравнение работает;
- summary работает;
- advice работает локально и не требует OpenAI;
- filters работают локально с последними результатами;
- export работает локально по последнему подбору и избранному;
- history работает и хранит только краткое описание последних подборов;
- психологическая поддержка работает готовыми безопасными ответами.

Safety/crisis-фразы не отправляются в OpenAI.

## Что сказать про Mini App

Mini App находится в папке `mini_app/` и сделан без frontend-фреймворков: HTML + CSS + JS.

Он работает через тот же backend endpoint:

```text
/api/universities
```

Локально Mini App открывается через:

```text
http://localhost:8000/miniapp
```

Для Telegram нужен публичный HTTPS URL через `WEBAPP_URL`.

В Mini App есть вкладки, локальные фильтры результатов, компактный итог подбора, визуальная таблица сравнения, избранное и переключение светлой/тёмной темы. Если Mini App открыт через Telegram, backend проверяет WebApp `initData`, Mini App показывает статус проверенной сессии, а избранное синхронизируется с Telegram-ботом. Сравнение и тема остаются локальными.

Для быстрой демонстрации в Mini App есть блок `Быстрый старт`: можно нажать готовый сценарий, например `Адыгея · 230 · IT · Бюджет`, и приложение само заполнит форму, запустит поиск и покажет toast-уведомление. Также в форме есть подсказки, очистка формы, понятные пустые состояния и toast-обратная связь при сохранении, сравнении и переключении темы.

Mini App не просто показывает список вузов, а формирует персональный план действий: сохранить варианты, сравнить их, проверить официальные сайты и выбрать безопасные, реалистичные и амбициозные направления. План строится локально по последнему подбору, избранному и сравнению, поэтому backend API не меняется.

Mini App также позволяет подготовить отчёт по подбору: параметры, найденные варианты, избранные вузы, сравнение и план поступления. Отчёт можно скопировать как текст или сохранить как PDF через системное окно печати браузера. PDF не генерируется программно, поэтому новые зависимости не нужны.

На защите можно показать, что Mini App переключается между светлой и тёмной темой без перезагрузки страницы и запоминает выбор после обновления.

Также можно показать визуальное сравнение: выбрать 2–3 вуза из результатов или избранного и открыть вкладку `Сравнение`, где Mini App сравнит программы по баллам, категории, типу обучения, стоимости, форме, сроку и сайту без повторного запроса к backend.

Можно показать синхронизацию избранного: сохранить вуз в Telegram-боте, открыть Mini App через `/webapp` и увидеть этот вуз во вкладке `Избранное`. Затем добавить вуз в Mini App и проверить, что он появился в Telegram-боте. Важно объяснить, что user_id не передаётся из клиента напрямую, а достаётся backend из проверенного Telegram WebApp `initData`.

Можно отдельно показать диагностику сессии: открыть Mini App в обычном браузере и увидеть `Локальный режим`, затем открыть через Telegram `/webapp` и увидеть, что сессия проверена. Это демонстрирует, что локальная разработка не ломается, но синхронизация включается только для проверенной Telegram-сессии.

Можно показать обратную связь: пользователь отправляет обращение через Telegram-бота или вкладку `Поддержка` в Mini App, получает номер заявки `AISH-0001`, а `/my_feedback` показывает последние обращения. Это демонстрационная локальная система: обращения сохраняются в `telegram_bot/storage/feedback.json`, без email, админ-панели и внешней базы.

### Почему проверка Mini App-сессии безопаснее

- `user_id` не принимается из query или body;
- Telegram WebApp `initData` проверяется на backend;
- HMAC считается через bot token, который не отправляется в Mini App;
- `hash` сравнивается через `compare_digest`;
- старые `auth_date` можно отклонять;
- без валидной сессии Mini App работает только локально и не получает доступ к избранному пользователя в боте.
- user_id для обращений Mini App берётся только из проверенного `initData`, а не из query/body клиента.

## Возможные вопросы преподавателя

### Где хранится база вузов?

Сейчас в `backend_stub/data/universities.json`. Это временная демонстрационная база.

### Что будет, если OpenAI ключа нет?

Бот продолжит работать через fallback. OpenAI не обязателен.

### Что делает backend?

Backend принимает параметры поиска, нормализует ввод, фильтрует `universities.json` и возвращает JSON-список программ.

### Зачем Mini App, если есть Telegram-бот?

Mini App показывает тот же подбор в визуальном веб-интерфейсе. Это удобно для демонстрации и дальнейшего развития проекта.

### Как работает сравнение?

Бот берёт последние результаты или избранные вузы и сравнивает минимальный балл, категорию, тип обучения, стоимость, предметы, форму и срок.

### Как работает история подборов?

После каждого поиска бот сохраняет краткую запись: регион, баллы, направление, тип обучения, количество результатов, категории и топ-3 варианта. Хранится максимум 5 последних записей в локальном JSON storage.

### Как работают советы по подбору?

Команда `/advice` берёт последний подбор, считает баланс безопасных, реалистичных и амбициозных вариантов и предлагает следующий шаг: сохранить вузы, сравнить варианты, проверить сайты, расширить регион, направление или тип обучения. OpenAI для этой функции не используется.

### Как работают фильтры?

Команда `/filters` берёт последний подбор и показывает фильтры: все, безопасные, реалистичные, амбициозные, бюджетные и платные варианты. Для сохранения после фильтра используется отдельное поле `active_results`, поэтому полный `last_results` остаётся доступен для `/summary`, `/history` и `/compare`.

### Как работает экспорт?

Команда `/export` берёт последний профиль, `last_results` и избранные вузы, формирует plain text отчёт и может отправить его сообщением или `.txt` документом. Export-файлы не сохраняются постоянно в проекте.

Во вкладке `Экспорт` Mini App отчёт строится локально по состоянию веб-интерфейса: последний подбор, избранное, сравнение и персональный план. Его можно скопировать или сохранить как PDF через печать браузера.

### Что такое safe/realistic/ambitious?

- safe - есть заметный запас по баллам;
- realistic - баллы близки к минимальному уровню;
- ambitious - немного не хватает, но вариант можно рассматривать как цель.

Это не гарантия поступления.

### Почему используется ngrok?

Telegram Mini App требует публичный HTTPS URL. Ngrok даёт временный HTTPS адрес для локальной демонстрации.

### Как заменить демонстрационную базу на настоящую?

Нужно заменить `backend_stub/data/universities.json` финальным файлом с той же структурой полей и запустить проверки:

```bash
python scripts/check_data.py
bash scripts/check_project.sh
```

Формат записей описан в `docs/DATA.md`. Если проверка проходит, логика бота и Mini App останется прежней, потому что они работают через единый backend API.
