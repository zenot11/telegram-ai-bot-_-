# Статус проекта “Аиша”

## Уже реализовано

- Telegram-бот на Python + aiogram 3;
- команды `/start`, `/menu`, `/help`, `/reset`, `/support`, `/search`;
- главное меню;
- FSM-подбор вузов;
- backend-заглушка;
- JSON-данные вузов;
- карточки результатов;
- хранение пользовательских данных локально;
- психологическая поддержка;
- safety-фильтр;
- реальная OpenAI-интеграция через официальный SDK `openai` и `AsyncOpenAI`;
- fallback без OpenAI;
- README;
- DEMO.md;
- архив старого C++ Telegram-модуля.

## Что является демонстрационным

- `backend_stub`;
- `universities.json`;
- проходные баллы;
- ссылки на сайты вузов;
- список направлений и регионов;
- локальное JSON-хранилище пользователя.

## Риски и ограничения

- данные вузов неофициальные;
- `backend_stub` нужен только для MVP;
- OpenAI работает только при наличии `OPENAI_API_KEY`;
- при отсутствии OpenAI используется fallback;
- `storage/user_data.json` локальный и не предназначен для продакшена;
- проект пока не содержит настоящей базы вузов.

## Текущая архитектура

Основной поток:

```text
Пользователь → Telegram Bot → backend_stub/API → JSON-данные → Telegram Bot → Пользователь
```

AI-слой:

```text
Telegram Bot → OpenAI API → AI-объяснение/поддержка
```

Safety-фильтр стоит до OpenAI: сообщения про самоповреждение, суицидальные мысли или опасность для себя обрабатываются локально и не отправляются в OpenAI.
