# Статус проекта “Аиша”

## Текущий статус

“Аиша” — расширенный демонстрационный прототип Telegram-сервиса для абитуриентов. Проект готов к показу преподавателю или команде: бот запускается, проводит пользователя через подбор вузов, показывает категории, сохраняет избранное, сравнивает варианты и открывает демонстрационный Mini App.

Сейчас JSON-база `backend_stub/data/universities.json` остаётся fallback по умолчанию. На 37 этапе добавлен PostgreSQL-режим для каталога вузов, а на 38 этапе backend получил справочники, расширенные фильтры и сортировки: `USE_POSTGRES=true` переключает `/api/universities` и справочники на PostgreSQL по `DATABASE_URL`, не меняя Telegram-бот и Mini App.

## База вузов

Данные в `backend_stub/data/universities.json` нужны для демонстрации логики и fallback-запуска без PostgreSQL. Перед сдачей файл можно заменить на финальную JSON-базу, сохранив структуру полей, либо загрузить SQL из `finalproj.zip` в PostgreSQL по `docs/POSTGRES.md`.

- `university`;
- `city`;
- `region`;
- `program`;
- `direction`;
- `subjects`;
- `min_score`;
- `type`;
- `price`;
- `url`;
- `study_form`;
- `duration`;
- `note`.

Если часть необязательных полей отсутствует, бот и Mini App должны показывать аккуратные fallback-значения, а не технические `None` или `null`.

Статус этапа 39: добавлен аудит совместимости JSON/PostgreSQL в `docs/DATA_COMPATIBILITY.md`, пользовательские карточки и экспорт скрывают пустые `subjects`, `price`, `duration`, `note`, а PostgreSQL-поля `year` и `faculty` выводятся как обычный полезный контекст. Диагностическое поле `source` остаётся в API и тестах, но не показывается в обычном UI.

Статус этапа 40: пользовательский поток переведён на PostgreSQL-first поведение при `USE_POSTGRES=true`, а JSON оставлен fallback. Mini App загружает справочники регионов, городов, направлений, форм обучения, типов конкурса и индивидуальных достижений из backend, показывает fallback-статус при недоступности backend и объясняет пустую выдачу через применённые фильтры. Backend добавил алиасы `Крым` -> `Республика Крым`, `Адыгея` -> `Республика Адыгея` и старое `IT` -> реальные SQL-направления. Карточки используют полное название вуза вместо технических `short_name` вроде `РЦТИ-26`, а `/api/achievements` отдаёт общий справочник достижений без привязки к конкретному вузу.

Статус этапа 41: PostgreSQL result card and score display polish. Значения `min_score=0/1` теперь остаются в API как исходные данные, но получают `score_is_valid=false`, `score_display="не указан"` и не используются для расчёта запаса или категорий safe/realistic/ambitious. Telegram-карточки, Mini App, экспорт, сравнение, summary и advice показывают контекст конкурса, года и пометки вместо `Запас: +279/+280`. Длинные карточки Mini App разделяют название вуза и программу, а short_name выводится только для нормальных аббревиатур, не для технических кодов.

Статус этапа 42: PostgreSQL mapping verification and label cleanup. Проверено, что `Региональный центр технологий и инженерии (...)` приходит из реальных строк `universities.name` supplemental seed, а не из ошибки JOIN/mapping. Обычная выдача скрывает synthetic/demo records по умолчанию, `include_synthetic=true`/`include_demo=true` оставлены для диагностики. Backend добавляет `financing_label`, `study_form_label`, `contest_label`; Mini App, Telegram-карточки, compare, export, summary и advice используют подписи `Финансирование`, `Форма обучения`, `Конкурс` и не показывают `Конкурс: бюджет` как дубль.

## Текущая архитектура

Основной поток:

```text
Пользователь → Telegram Bot → backend_stub/API → JSON/PostgreSQL → Telegram Bot → Пользователь
```

Mini App:

```text
Mini App → backend_stub/API → JSON/PostgreSQL → Mini App
```

AI-слой:

```text
Telegram Bot → OpenAI API → AI-объяснение/поддержка
```

Safety-фильтр стоит до OpenAI: сообщения про самоповреждение, суицидальные мысли или опасность для себя обрабатываются локально и не отправляются в OpenAI.

## Риски и ограничения

- данные вузов демонстрационные и не являются официальными проходными баллами;
- категории подбора помогают ориентироваться, но не гарантируют поступление;
- OpenAI работает только при наличии `OPENAI_API_KEY`;
- при отсутствии OpenAI используется fallback;
- `WEBAPP_URL` может быть пустым, тогда кнопка `Mini App` в меню работает как текстовая подсказка без WebApp-открытия;
- для открытия Mini App внутри Telegram нужен публичный HTTPS URL;
- в обычном браузере Mini App работает в local mode, избранное хранится только в `localStorage`; синхронизация с ботом работает только после проверенной Telegram WebApp-сессии;
- `storage/user_data.json` локальный и не предназначен для постоянного общего хранения данных;
- история подборов хранится локально в JSON и ограничена последними 5 записями;
- export-файлы не хранятся постоянно и формируются только для отправки пользователю;
- обращения хранятся локально в `telegram_bot/storage/feedback.json`, файл не попадает в Git;
- JSON-базу перед сдачей нужно проверить через `python scripts/check_data.py`;
- PostgreSQL-режим требует настроенной БД и `DATABASE_URL`;
- продакшен-деплой не входит в текущий этап.
