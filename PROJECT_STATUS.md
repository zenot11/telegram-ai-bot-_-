# Статус проекта “Аиша”

## Текущий статус

“Аиша” — расширенный демонстрационный прототип Telegram-сервиса для абитуриентов. Проект готов к показу преподавателю или команде: бот запускается, проводит пользователя через подбор вузов, показывает категории, сохраняет избранное, сравнивает варианты и открывает демонстрационный Mini App.

Сейчас используется временная демонстрационная база `backend_stub/data/universities.json`. Финальная база вузов будет подставлена перед сдачей. Структура проекта уже подготовлена к замене демонстрационного JSON на финальные данные без переписывания основной логики.

## Временная база вузов

Данные в `backend_stub/data/universities.json` нужны для демонстрации логики. Перед сдачей файл можно заменить на финальную базу вузов, сохранив структуру полей:

- `university`;
- `city`;
- `region`;
- `program`;
- `direction`;
- `subjects`;
- `min_score`;
- `type`;
- `price`;
- `url`;
- `study_form`;
- `duration`;
- `note`.

Если часть необязательных полей отсутствует, бот и Mini App должны показывать аккуратные fallback-значения, а не технические `None` или `null`.

## Текущая архитектура

Основной поток:

```text
Пользователь → Telegram Bot → backend_stub/API → JSON-данные → Telegram Bot → Пользователь
```

Mini App:

```text
Mini App → backend_stub/API → JSON-данные → Mini App
```

AI-слой:

```text
Telegram Bot → OpenAI API → AI-объяснение/поддержка
```

Safety-фильтр стоит до OpenAI: сообщения про самоповреждение, суицидальные мысли или опасность для себя обрабатываются локально и не отправляются в OpenAI.

## Риски и ограничения

- данные вузов демонстрационные и не являются официальными проходными баллами;
- категории подбора помогают ориентироваться, но не гарантируют поступление;
- OpenAI работает только при наличии `OPENAI_API_KEY`;
- при отсутствии OpenAI используется fallback;
- `WEBAPP_URL` может быть пустым, тогда Mini App-кнопка в меню не показывается;
- для открытия Mini App внутри Telegram нужен публичный HTTPS URL;
- избранное Mini App хранится только в `localStorage` текущего браузера;
- `storage/user_data.json` локальный и не предназначен для постоянного общего хранения данных;
- история подборов хранится локально в JSON и ограничена последними 5 записями;
- продакшен-деплой не входит в текущий этап.

## Примечание по Mini App

На 14 этапе изменён только внешний вид Mini App: светлый интерфейс, hero-блок, карточка “Привет, я Аиша!”, тёмно-синие кнопки и светлые карточки результатов. Логика подбора, API `/api/universities`, параметры запроса и структура данных не изменялись.

На 15 этапе Mini App по логике не менялся. Полировка касалась Telegram-бота: формат карточек, итог подбора, `/summary`, `/demo`, `/help` и BotFather-команд.

На 22 этапе Mini App стал ближе к полноценному приложению внутри Telegram: добавлены вкладки, локальные фильтры результатов, избранное через `localStorage`, итог подбора и короткие советы. Backend API, структура `universities.json` и логика Telegram-бота не менялись.
